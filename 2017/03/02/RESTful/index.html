RESTful

RESTful应该算是目前比较流行的互联网网络架构了，服务器开发可以以REST来开发一套网页端和移动端都能使用的api。

REST主要是三个东西：资源、表现层和状态转化。

资源：就是服务器提供的一个东西或者一段信息，比如一段音频，视频，或者一个用户信息。资源通过url来描述。

表现层：资源通过哪种方式来展现，图片通过jpg和png来描述，用户信息通过json来展现。通过http请求的头文件的Accept和Content-type用来描述表现层。比如Content-type="application/json"就是通过json来展现资源。

状态转化：客户端想要对服务端的数据做一些改变，比如修改一些数据，删除订单，需要请求一个接口，调用接口的url，但是根据REST框架，url是描述一个资源，是一个东西，而不是一个行为（比如删除订单），所以需要有状态转化去确定这个操作是什么。动词有get（获取资源），push（新建资源），put（修改资源），delete（删除资源）。这个状态转换也是放在http头文件中的
