我们通过最简单的调用一个接口来分析Retrofit流程。

Retrofit通过建造者模式创建实例，再通过动态代理进行一个接口的调用，在动态代理中Retrofit会将接口方法解析并生成一个OkhttpCall，并使用Okhttp进行网络请求。

1、new Retrofit.Builder()

总的来说，new Retrofit.Builder()创建了一个拥有初始化Retrofit方法的Android类

Platform.get()会一直调用到findPlatform()

这里会先判断是否是安卓系统，另外会判断是否是java8，我们只需要分析new Android()，Android()类中会有一些初始化的方法，也定义了一个线程执行器MainThreadExecutor()。具体代码等到调用到的时候再依次贴出来。

2、.baseUrl("https://api.github.com/")

baseUrl解析了baseUrl，将baseUrl解析为各个部分，scheme、host、port，放到生成的HttpUrl实例中。

3、.build()

build方法将Retrofit默认的CallAdapter和Converter创建好，并放入两个队列中。目前CallAdapter有CompletableFutureCallAdapterFactory、DefaultCallAdapterFactory。Converter有BuiltInConverters、OptionalConverterFactory。

4、.create(Domain.class)

我们这里的注释其实分析了上个步骤，

1、.create()，范围在create方法的作用域中。这里总结一下先是对接口类做了校验，再通过用户配置的结果来看是否对接口类的方法的配置注解进行校验。最后创建动态代理的实例。

2、invoke()就是回调里的作用域，总结就是对调用的方法进行校验，方法所在类的校验，接口default方法的校验。最后进入loadServiceMethod(method).invoke(args != null ? args : emptyArgs);,也就进入了Retrofit最重要的地方了。

所以当调用listRepos("aaa")就进入了loadServiceMethod(method).invoke()

这里是关于HttpServiceMethod.parseAnnotations(retrofit, method, requestFactory)，这里我们分析完成之后，我们再回到最开始，动态代理的地方。

这部分是获取CallAdapter的流程。

至此，最简单的Retrofit的流程就ok了。

我们来探讨一下这个问题，Retrofit有没有必要做单例。

我们来看看Retrofit初始化的流程。new Retrofit.Builder().baseUrl().build().create()

这个链是怎么样的呢？Builder <- retrofit <- T，T就是我们方法的接口，而我们分析知道，Retrofit实例里会保存每一个调用过的方法的HttpServiceMethod，所以我们按道理来讲应该是单例Retrofit就ok了，让我们试验一下。
