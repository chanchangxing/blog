递增的三元子序列

给定一个未排序的数组，判断这个数组中是否存在长度为 3 的递增子序列。

数学表达式如下:

如果存在这样的 i, j, k, 且满足 0 ≤ i < j < k ≤ n-1，
使得 arr[i] < arr[j] < arr[k] ，返回 true ; 否则返回 false 。
说明: 要求算法的时间复杂度为 O(n)，空间复杂度为 O(1) 。

我的思路自己感觉已经是相当精彩了，设置两个值（M1，M2），M1保存最小值，M2保存目前最合理的第二个值，因为我们只需要确认了第一个和第二个值，那么只需要保存第二个值就ok了。
那么保存的规则就是这样的，M1保存数组里最小的值，M2通过M1和当前header指向的值进行比较，如果该值比M2要大，那么就有一个
递增的三元子序列，否则看是否大于M1小于M2，这样的话就将他设置为新的M2,最后再将最后一个数和M2进行比较，如果最后一个值大于M2就是递增的三元子序列。

最佳答案就是我这个思路好吧。无敌！
