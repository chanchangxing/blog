呆萝卜面试总结

呆萝卜没有android技术面，只有技术老大的面试。

  1. 关于兔波波销售的技术难点。
  2. 关于一个稳定的App需要做些什么。

  3. 关于今后的发展。

  4. 关于第一个问题，已经不是我回答的不好了，而是做项目根本就没有什么难点啊，唯一的难点就是RN给我造成的问题，那也是对RN和JS的不熟悉。

  5. 第二个问题我自认为回答的还是比较好的，虽然总感觉是自己理解错了问题，但也是将自己的对安卓开发的理解表达了一下。也有些拉掉的比如说app安全，这里就再总结一下吧。从外到里
    • app开启速度，好像是举例了微信和支付宝
    • 页面适配、机型适配、刘海屏适配。
    • 权限通知
    • 断网的情况下数据的缓存。
    • apk大小，也涉及到混淆，安全性
    • app安全性，分为app防逆向，app网络安全、app本地数据安全。
    • 热更新
    • app监控
    • 日志系统
    • 保活
  6. 今后的发展的话我回答的是大前端方向，但我觉得我在为什么选择大前端方向这个话题上聊的太久了，而且原因是因为后端没时间学了。我觉得应该要谈的是前端方向的优势，里面也有很多东西可以挖掘。
